# BC Cycling + Supabase

Projeto migrado para front-end puro, sem PHP.

## Como configurar

1. Crie as tabelas no Supabase:

```sql
create table public.workouts (
  id bigint generated by default as identity primary key,
  start_time timestamptz not null default now()
);

create table public.workout_data (
  id bigint generated by default as identity primary key,
  workout_id bigint not null references public.workouts(id) on delete cascade,
  time integer not null,
  power integer,
  cadence integer,
  heart_rate integer,
  speed numeric(10,2)
);

create index workout_data_workout_id_idx
  on public.workout_data (workout_id);

create index workout_data_workout_id_time_idx
  on public.workout_data (workout_id, time);

create table public.workout_music (
  id bigint generated by default as identity primary key,
  workout_id bigint not null references public.workouts(id) on delete cascade,
  title text not null,
  artist text,
  duration_seconds integer
);

create table public.workout_music_blocks (
  id bigint generated by default as identity primary key,
  workout_id bigint not null references public.workouts(id) on delete cascade,
  music_id bigint not null references public.workout_music(id) on delete cascade,
  start_time integer not null,
  end_time integer not null,
  intensity text not null,
  position text not null check (position in ('sentado', 'em_pe')),
  load_label text not null,
  check (end_time >= start_time)
);

create index workout_music_workout_id_idx
  on public.workout_music (workout_id);

create index workout_music_blocks_workout_id_idx
  on public.workout_music_blocks (workout_id);

create index workout_music_blocks_music_id_idx
  on public.workout_music_blocks (music_id);
```

2. Ative RLS e crie as politicas se for usar `anon key` no navegador:

```sql
alter table public.workouts enable row level security;
alter table public.workout_data enable row level security;
alter table public.workout_music enable row level security;
alter table public.workout_music_blocks enable row level security;

create policy "Allow read workouts"
on public.workouts
for select
to anon, authenticated
using (true);

create policy "Allow insert workouts"
on public.workouts
for insert
to anon, authenticated
with check (true);

create policy "Allow read workout_data"
on public.workout_data
for select
to anon, authenticated
using (true);

create policy "Allow insert workout_data"
on public.workout_data
for insert
to anon, authenticated
with check (true);

create policy "Allow read workout_music"
on public.workout_music
for select
to anon, authenticated
using (true);

create policy "Allow insert workout_music"
on public.workout_music
for insert
to anon, authenticated
with check (true);

create policy "Allow read workout_music_blocks"
on public.workout_music_blocks
for select
to anon, authenticated
using (true);

create policy "Allow insert workout_music_blocks"
on public.workout_music_blocks
for insert
to anon, authenticated
with check (true);
```

3. A configuracao do Supabase ja foi preenchida em `js/supabase-config.js`.

4. Abra `index.html` direto no navegador.

## Fluxo

- `index.html`: inicia treino e mostra historico/grafico do treino mais recente
- `workout.html?id=1`: adiciona amostras manualmente e atualiza grafico/tabela
